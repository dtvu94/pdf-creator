# PDF Creator

A full-stack PDF generation platform with a visual template editor and a standalone Java microservice for PDF processing.

## Architecture

| Service | Stack | Port | Description |
|---------|-------|------|-------------|
| **pdf-creator** | Next.js 16, React 19, TypeScript | 3000 | WYSIWYG template editor + PDF generation via React PDF |
| **pdfa-generator** | Java 17, Apache PDFBox 3, BouncyCastle | 8090 | Standalone PDF generation, PDF/A conversion, encryption, signing, merging, splitting, text extraction, metadata extraction, text & image watermarks |

Both services can generate PDFs independently from the same template JSON format. `pdf-creator` uses React PDF (JavaScript), while `pdfa-generator` uses Apache PDFBox (Java).

## Quick Start

### With Docker Compose (recommended)

```bash
docker compose up --build
```

This starts both services:

- **pdf-creator** at `http://localhost:3000` (editor UI + API)
- **pdfa-generator** at `http://localhost:8090` (processing API)

### Without Docker

**pdf-creator:**

```bash
cd pdf-creator
npm install
npm run dev
```

**pdfa-generator** (requires Java 17+):

```bash
cd pdfa-generator
mvn clean package
BUNDLED_FONT_DIR=../pdf-creator/public/fonts java -jar target/pdfa-generator-1.0.0.jar
```

## Usage

### Editor UI

Open `http://localhost:3000` to access the template editor. Choose from 18 built-in templates or start blank. Edit visually, then export as PDF with optional placeholders, CSV data, metadata, password protection, PDF/A conversion, digital signatures, image compression, and PDF bookmarks. Templates can be shared via URL or clipboard.

### API — pdf-creator (Next.js)

```bash
# Generate PDF from a template
curl -X POST http://localhost:3000/api/generate-pdf \
  -F "template=@template.json" \
  -F "placeholders=@values.json" \
  -o output.pdf
```

### API — pdfa-generator (Java/PDFBox)

```bash
# Generate PDF from the same template
curl -X POST http://localhost:8090/api/generate \
  -H "Content-Type: application/json" \
  -H "Accept: application/pdf" \
  -d '{"template": {...}, "placeholders": {"key": "value"}}' \
  -o output.pdf

# PDF/A conversion
curl -X POST http://localhost:8090/api/convert-to-pdfa \
  -H "Content-Type: application/json" \
  -d '{"pdf": "<base64>", "part": 2, "conformance": "B"}'

# Encrypt, sign, merge, split, extract text/metadata, watermark...
# See pdfa-generator/README.md for full API reference.
```

## Sample PDFs

The `generate-sample-pdfs.sh` script generates PDFs from all 18 built-in templates using both services, allowing side-by-side comparison.

**Prerequisites:** Both services must be running.

```bash
# Start services first, then:
bash generate-sample-pdfs.sh
```

Output goes to `sample-pdf/`:

```bash
sample-pdf/
  invoice-nextjs.pdf      # Generated by pdf-creator (React PDF)
  invoice-pdfa.pdf        # Generated by pdfa-generator (PDFBox), PDF/A-2b
  report-nextjs.pdf
  report-pdfa.pdf
  ...
```

The `-pdfa.pdf` files are post-processed through `/api/convert-to-pdfa` so they
are PDF/A-2b compliant by default. Override with `PDFA_PART=1|2|3` and
`PDFA_CONFORMANCE=A|B|U`, e.g. `PDFA_PART=3 bash generate-sample-pdfs.sh`.

## Project Structure

```bash
pdf-creator/              # Next.js web application
  app/                    # Pages and API routes
  components/             # Editor UI components
  lib/                    # Renderers, templates, utilities
  public/fonts/           # Bundled TTF fonts (32 files)
  public/samples/         # Sample data per template

pdfa-generator/           # Java microservice
  src/main/java/
    com/pdfcreator/pdfa/
      App.java            # HTTP server + endpoints
      PdfAConverter.java  # PDF/A conversion + validation
      PdfEncryptor.java   # Password encryption
      PdfSigner.java      # Digital signatures
      PdfMergerSplitter.java
      PdfTextExtractor.java
      PdfMetadataExtractor.java
      PdfWatermarker.java # Text & image watermarks
      generator/          # PDF generation engine
        PdfGenerator.java
        PdfElementRenderer.java
        PdfPageLayout.java
        PdfTextLayout.java
        PdfShapeRenderer.java
        PdfFontManager.java
  fonts/                  # Bundled TTF fonts (copied for Docker)

docker-compose.yml        # Orchestrates both services
generate-sample-pdfs.sh   # Generates comparison PDFs
```

## Presentation PDF

A Python script generates a 12-page overview presentation of the platform for internal audiences.

**Prerequisites:** Docker (or Python 3 with `reportlab`).

```bash
# With Docker (no local Python dependencies needed)
docker run --rm -v "$(pwd)":/output python:3.12-slim \
  bash -c "pip install reportlab -q && python3 /output/generate_presentation.py"

# Or with local Python
pip install reportlab
python3 generate_presentation.py
```

Output: `presentation.pdf` in the project root.
